开发工具集静态网页 - 详细开发文档

项目概述

开发一个纯静态的开发工具集网页，作为浏览器主页使用。该网页采用配置驱动的方式，所有工具信息在index.html中定义，支持本地和远程工具页面。

技术栈

• HTML5

• Tailwind CSS（通过CDN引入）

• Vue 3（通过CDN引入）

• 纯静态实现，无构建工具

项目结构


dev-tools/
├── index.html              # 主入口页面，包含所有配置
├── json-formatter.html     # JSON格式化工具页面
├── encoder-decoder.html    # 编解码工具页面
├── api-tester.html         # API测试工具页面
├── base64-converter.html   # Base64转换工具页面
├── timestamp-converter.html # 时间戳转换工具页面
└── ... (其他工具页面)


功能设计

主页面布局设计 (index.html)

整体布局

• 采用左右分栏布局

• 左侧为固定宽度的工具导航栏

• 右侧为自适应宽度的工具内容区域

左侧导航栏

• 显示所有已配置的工具列表

• 每个工具项包含图标和名称

• 支持工具分类分组显示

• 当前选中工具高亮显示

• 支持工具搜索和筛选功能

右侧内容区域

• 通过iframe嵌入选中的工具页面

• 显示当前使用工具的名称

• 提供工具切换和历史记录功能

配置系统设计

配置结构

在主页面中定义一个JavaScript配置对象，包含所有工具信息：
const toolConfig = {
  // 应用设置
  app: {
    name: "开发工具集",
    version: "1.0.0"
  },
  
  // 工具分类
  categories: [
    {
      id: "format",
      name: "格式化工具",
      icon: "图标URL"
    },
    {
      id: "encode",
      name: "编解码工具", 
      icon: "图标URL"
    },
    // 更多分类...
  ],
  
  // 工具列表
  tools: [
    {
      id: "json-formatter",
      name: "JSON格式化",
      category: "format",
      icon: "https://example.com/icons/json.svg",
      url: "json-formatter.html",
      description: "格式化、验证和美化JSON数据"
    },
    {
      id: "encoder-decoder",
      name: "编解码工具",
      category: "encode", 
      icon: "https://example.com/icons/encode.svg",
      url: "encoder-decoder.html",
      description: "Base64、URL等多种编码解码"
    },
    // 更多工具...
  ]
};


配置特性

• 支持本地HTML文件路径

• 支持远程工具URL

• 支持工具分类管理

• 包含工具描述信息

• 支持图标远程URL

详细实现方案

1. 主页面实现

HTML结构

• 使用语义化HTML5标签

• 采用Tailwind CSS类名进行样式设计

• 实现响应式布局，适配不同屏幕尺寸

Vue应用结构

• 使用Vue 3组合式API

• 实现工具列表渲染和选择逻辑

• 管理当前选中工具状态

• 实现工具搜索和筛选功能

核心功能模块

• 工具列表组件：渲染工具导航栏

• 内容区域组件：管理iframe加载和显示

• 搜索组件：实现工具搜索功能

• 分类筛选组件：按分类显示工具

部署和使用说明

部署要求

• 任何静态文件服务器

• 支持现代浏览器的Web环境

使用流程

1. 将所有HTML文件部署到Web服务器
2. 在浏览器中打开index.html
3. 从左侧导航栏选择需要的工具
4. 在右侧内容区域使用工具功能

浏览器兼容性

• Chrome 60+

• Firefox 55+

• Safari 12+

• Edge 79+

扩展和维护

添加新工具

1. 创建工具HTML页面
2. 在index.html的配置中添加工具信息
3. 测试工具功能

自定义配置

• 修改toolConfig对象添加、删除或编辑工具

• 调整分类和工具顺序

• 更新图标和描述信息

样式定制

• 通过修改Tailwind CSS类名调整样式

• 添加自定义CSS覆盖默认样式

• 调整颜色和布局参数

注意事项

1. 所有工具页面应保持功能独立
2. 避免工具页面间的样式冲突
3. 确保远程资源可用性
4. 注意工具页面的加载性能
5. 提供必要的错误处理和用户反馈
